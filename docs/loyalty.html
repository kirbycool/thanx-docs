Thanx Loyalty API API Reference

This documentation describes the endpoints supported by the Thanx Loyalty API.

The purpose of this API is to provide a way to retrieve a user's rewards and apply discounts to their basket. An online ordering platform with a Thanx merchant as a customer can use this API to allow Thanx users to use their discounts when making purchases online. The GET /account endpoint provides a user's available rewards. The POST /baskets endpoint computes a discount and marks rewards as used in the Thanx system. Integrators are expected to apply the discount to the basket.

Paths

Loyalty

Retrieve user account information, including rewards

GET /account

This endpoint allows the retrieval of a user's rewards. By default, only rewards available to use at the current time are returned.

Authorization: string
in header

All Thanx Loyalty API endpoints are protected and must be authorized via end user access tokens. These access tokens can be retrieved through an integration with Thanx SSO. The format of the header should be: Bearer access_token

Content-Type: string
in header

The only accepted value is application/json

Accept: string
in header

The Accept header specifies which version of the Thanx API that should be used. The current version is v1 for the vendor thanx. This header is required for every request. The value should be application/vnd.thanx-v1+json. Thanx will notify you when a new API version is available.

Merchant-Key: string
in header

The Merchant key header identifies the merchant in Thanx. The value for this key will be provided to you by Thanx and is the same in production and sandbox.

location_uid: string
in query

The location identifier; this is what your system uses to identify a location. Providing this information allows Thanx to return rewards that can be used at this location.

Loyalty

Send basket information to Thanx for reward redemption

POST /baskets

This endpoint is used to notify Thanx of the states that a basket can be in and receive reward redemption information in return. A basket can be in one of the following states: [checkout, placed, billed, completed, voided, refunded]. checkout refers to any point before the order is placed. Any time the user updates their basket or rewards, Thanx expects to receive a request to check if discount information should be updated. placed refers to when an order has been submitted to the system and a user can no longer modify their order through the website. billed refers to when the order has been trasmitted to the POS and the user's credit card has been charged. completed refers to when the order has been made and transferred to the customer. voided refers to an order that got cancelled after entering the placed state and before entering the billed state. refunded refers to an order being refunded / cancelled any time after it enters the billed state.

Thanx will mark a reward as used and grant loyalty reward progress when the reward enters the billed state.

The Reward Identifier

The value of the discount, present for 'amount' and 'percent' types

minimum: number

The minimum spend for this reward, if applicable

maximum: number

The maximum discount possible for this reward, if applicable

label: string

The reward description

state: string

The state of the reward: 'redeemable' or 'unredeemable'

type: string

The type of reward: 'amount' or 'percent' or 'item'

products: string[]

POS Identifiers for items this discount applies to, if applicable

string

Basket: object

id: string

An identifier for the basket - should be the same for a basket as it changes state

state: string checkout, placed, billed, completed, voided, refunded

The state of the basket

order_timestamp: string

When the order is wanted in (ISO 8601 format). If this timestamp is not provided, Thanx assumes the order is wanted ASAP. Providing this timestamp allows Thanx to determine if a reward will be valid in the future (for example, for a weekend only promotion where the user orders in advance)

location_uid: string

The location identifier; this is what your system uses to identify a location. Providing this information allows Thanx to apply location restricted rewards.

rewards: string[]
